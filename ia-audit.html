Key Migration Decisions

  • Navigation model change: Shifting from topic-based ("About FMNR", "Where We Work") to audience-based ("Partner", "Learn", "Adopt") to better serve distinct user groups
  • "Partner with Us" evolves: Existing partnerships page becomes the top-level "Partner" section with sub-pages for each audience (faith, NGO, government, funders)
  • "Get Involved" removal: Donate/Volunteer/Support Us pages removed — fundraising handled through World Vision main site; partnership now has its own top-level section
  • Country pages preserved: All 11 country pages plus the "Country Specific" landing page migrate into "Adopt → Country Examples" with enhanced data and storytelling
  • Resources consolidation: Publications, Videos, and Toolkits merge into a single filterable Resources Hub with taxonomy-based browsing
  • E-Course kept: Existing e-course page migrates into the new "Adopt" section as a standalone learning pathway
  • "FMNR Fundamentals" + "What is FMNR?" merge: Two overlapping explainer pages consolidate into "Learn → FMNR Fundamentals"
  • "FMNR and World Vision" moves: Existing org page becomes "About → FMNR Hub & World Vision"
  • "Learn" section is new: Combines content from "About FMNR", "FMNR Fundamentals", "How FMNR Works", and "Research" into a coherent educational pathway
  • Impact Dashboard is new: A data-driven interactive page replacing the static "Impact & Evidence" section with live metrics and progress tracking
  • Blog → Stories: Blog archive migrates but rebrands as "Stories" under Impact, with enhanced card-based browsing and tagging
  • GGW Webinar removed: One-off event page is outdated — content can be archived or referenced in blog
  • Search is new: Dedicated search results page with faceted filtering — does not exist on current WordPress site
  • URL strategy: All old URLs (including ?page_id= permalinks) should 301 redirect to new equivalents — create redirect map during build phase
  • SEO considerations: Preserve existing meta descriptions and H1 content where pages are kept; update canonical URLs for merged pages
